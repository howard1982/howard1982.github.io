fstab 格式說明

  • <file systems> - 要挂载的分区或存储设备
  • <dir> - <file systems>的挂载位置。
  • <type> - 要挂载设备或是分区的文件系统类型，支持许多种不同的文件系 如：ext4 xfs
  • <options> - 挂载时使用的参数，注意有些mount 参数是特定文件系统才有的
  • <dump> dump 工具通过它决定何时作备份. dump 会检查其内容，并用数字来决定是否对这个文件系统进行备份。 允许的数字是 0 和 1 。0 表示忽略， 1 则进行备份。大部分的用户是没有安装 dump 的 ，对他们而言 <dump> 应设为 0。
  • <pass> fsck 读取 <pass> 的数值来决定需要检查的文件系统的检查顺序。允许的数字是0, 1, 和2。 根目录应当获得最高的优先权 1, 其它所有需要被检查的设备设置为 2. 0 表示设备不会被 fsck 所检查。

incremental back 增量备份

以文件为最小的单位。 利用 linux 的硬链接 (Hard link) 特性（多个link不会增加磁盘，删除某个不会影响其他的）空间来实现。

基本操作

rsync 命令

  • --link-dest 增量備份

说明

  1. 通过 ls -i 命令 可以查看文件的编号。 如果编号一样则表示是 硬链接。或用 stat [file] 命令查看更详细的信息

sar 常用命令 & 查看系統流量

发表于 2017-12-01 更新于 2021-06-21

安裝

yum install sysstat

說明

sar -n DEV 2 4

显示网络接口信息 命令后面2 4 意思是：每2秒钟取1次值，取4次。

  • IFACE：LAN接口
  • rxpck/s:每秒钟收到数据包的数量.
  • txpck/s:每秒钟发送数据包的数量.
  • rxkB/s:每秒钟接收的字节(KB).
  • txkB/s:每秒钟发送的字节(KB).
  • rxcmp/s:每秒收到的压缩包的数量
  • txcmp/s:每秒发出的压缩包的数量
  • rxmcst/s:每秒收到的广播包的数量

常用命令

  • sar 5 5 // CPU和IOWAIT统计状态
  • sar -p 1 3 // cpu 狀態
  • sar -n DEV 2 4 // 網絡

Apache MPM

发表于 2021-06-08

prefork MPM

Implements a non-threaded, pre-forking web server
prefork模式可以算是很古老但是非常稳定的Apache模式。

  • 缺点：占用内存多。不擅长处理高并发请求。

worker MPM

Multi-Processing Module implementing a hybrid multi-threaded multi-process web server
worker模式使用了多进程和多线程的混合模式

-优点：占据更少的内存，高并发下表现更优秀。

event MPM

A variant of the worker MPM with the goal of consuming threads only for connections with active processing
最新的模式，worker的一个变体。它和worker模式很像。最大的区别在于，它解决了keep-alive 场景下，长期被占用的线程的资源浪费问题

选择

要求 更高伸缩性的站点可以选择使用线程的 MPM，即 worker 或 event； 需要可靠性或者与旧软件兼容的站点可以使用 prefork。
